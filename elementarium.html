Elementarium.

What if there was a more engaging way to reach consumers that would increase brand awareness. Combining experimental browser-based 3D and Interaction Design to create a more engaging advertisement. There has been some exploration into VR advertisements with the a VR headset such as oculus but that requires consumers to either own a VR headset or to go to a physical location. With my solution I wanted to create a phone based VR that is accessible to everyone, wherever they are. Also moving from a Graphical User Interface to a Natural User Interface.

  • Client: Experimental Senior Class Project
  • Role: UX Designer and three.js Developer
  • Timeline: JAN - FEB 2014 ( 3 Months )

Setting goals.

The project goals were to rethink an interaction in terms of a particular UI problem or scenario, work through ideas and solutions that are radically different, and create something that fundamentally changes how we think about our interactions with contemporary technologies.

Project objective.

To achieve the project goal I created four objectives:
1. Research and be inspired by what currently exists.
2. Sketch interactions, ideas and feelings using methods of layering and not erasing.
3. Design the workflow and interactions.
4. Finally create an interactive prototype with Three.js.

User research.

Based on the project goals most of the research portion was focused on researching what we were interested in as well as what other people were doing in the experimental field. So for me that meant researching webGL, VR, AR, interfaces, instalations, and interesting interactions.

Research.

In my research I looked at multiple projects, some profesional and some personal projects. From this research I became interested in WebGL, VR, and using non standard interactions.

UX design.

The user experience design of this project was largely focused on designing the interactions in the 3D space. I needed to make sure that everything was effortless and intuitive. As well as intergrating natural user interface interactions.

Sketching.

With my sketches I worked through the layout and structure of the app as well as some interactions in a low fidelity. Then I took the sketches to high fidelity so I could fully understand the structure.

Features and functions.

Listing out all of my features and functions so that I know what I'm working with for the workflow.

Workflow.

Working through how I envision the user "walking" through the interaction.

Interaction design.

As I said above I was trying to make all interactions intuitive and effortless, as well as intergrating NUI interations.

Inspiration.

With the branding I started by trying to figure out the feeling that I would want the user to feel when using the app. I decided to convey Human, Artistic, Friendly, I wanted the user to feel safe and creative. Then I started exploring colors and fonts that conveyed this feeling. I originally started with an orange but switched to a teal to make it feel more energetic and fun. For the logo font I went with Poppins and for app / site copy I went with Open Sans, both fonts are humanist fonts so they look more friendly and human.
